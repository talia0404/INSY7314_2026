# 🎮 GameVault

Throughout this semester, we will progressively build **GameVault**, a secure full-stack web application for managing and discovering video games.

Rather than creating several small projects, we will continuously improve **one application** as we learn new concepts and technologies. Each learning unit will introduce new features and security measures that expand the application.

By the end of the semester, GameVault will be a modern, secure web application that demonstrates industry-standard software development practices.

---

## 🕹️ What is GameVault?

GameVault is a secure platform where users can browse video games, manage their personal collections, write reviews, and discover new titles.

Different users will have different permissions within the system, ensuring that only authorised users can perform specific actions.

Throughout the semester, we'll transform GameVault from a simple API into a fully functional MERN application with testing, security, containerisation, and DevSecOps.

---

# 🚀 How GameVault Will Grow

### 📚 Learning Unit 1 – Foundations

We'll build the foundation of our application by learning how web applications work.

By the end of this learning unit, we'll have:

* Our first Node.js and Express application
* A working web server
* API routes and endpoints
* JSON responses
* Input validation
* Environment variables
* Basic security principles

---

### 🔧 Learning Unit 2 – Secure Backend Development

We'll transform our basic server into a structured backend API.

New features include:

* CRUD operations for games
* Express routing
* Controllers
* Middleware
* HTTPS
* Helmet
* Content Security Policy (CSP)
* CORS
* API testing with Postman

At this stage, GameVault will become a professional-looking backend application.

---

### 🔐 Learning Unit 3 – Authentication & Data Security

We'll introduce users and secure access to the application.

New features include:

* MongoDB database
* User registration
* Login
* Password hashing
* JSON Web Tokens (JWT)
* Role-Based Access Control (RBAC)
* Protected API routes
* Rate limiting

Different users will have different permissions, ensuring that only authorised users can manage games and reviews.

---

### 💻 Learning Unit 4 – Frontend Development

Now we'll bring GameVault to life by building the user interface using React.

Features include:

* User registration and login
* Browse all games
* View game details
* Add games to personal collections
* Submit reviews
* Manage your own profile
* Admin dashboard

The frontend will securely communicate with the backend API.

---

### 🧪 Learning Unit 5 – Testing & Containerisation

We'll improve the quality and reliability of our application.

We'll learn how to:

* Write automated tests
* Test APIs using Postman and Newman
* Test React components
* Improve code quality using ESLint
* Containerise the application using Docker and Docker Compose

By the end of this unit, GameVault will be easier to test, deploy, and maintain.

---

### ⚙️ Learning Unit 6 – DevSecOps & Deployment

Finally, we'll prepare GameVault for deployment using modern software engineering practices.

We'll implement:

* GitHub Actions
* CI/CD pipelines
* Static code analysis
* Automated testing
* Application logging
* Monitoring
* Secure deployment practices

This is how professional software teams build, test, and maintain applications.

# 🎯 Why Are We Building GameVault?

GameVault is our **learning project**.

It allows us to practise modern software development techniques in a realistic environment while learning how to build secure, scalable applications.

The concepts and skills you develop while building GameVault will prepare you to design and implement your own secure application in the module assessments.

> 💡 **Remember:** GameVault is a demo application. It demonstrates the concepts covered in class, but your assessments will require you to apply these concepts independently to a different problem domain.
